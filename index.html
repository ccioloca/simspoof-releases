SimSpoof takes a route file and plays it as real movement on an iPhone Simulator, an Android emulator, or a real iPhone — as if the phone were actually travelling. Watch the dot move, control the speed, and see your app react. No walking the route.

Getting started

SimSpoof is a Mac app (Windows too, minus the iPhone-Simulator part).

1 · Install

  • Download the latest build from the Releases page.
  • Open it. It updates itself when a new version ships.
  • Any running simulator, emulator, or connected iPhone shows up automatically — and is selected for you.

2 · Have a phone running

  • iPhone Simulator — boot one from Xcode (or Simulator.app).
  • Android emulator — start one from Android Studio.
  • Real iPhone — plug it in with Developer Mode on; it appears as another selectable phone.
  • To see your app's own debug logs, run it as a development build.

How to use it

Four steps from launch to watching your app react.

  1. Pick your phones

    Every simulator, emulator, or connected iPhone that's running appears in the sidebar and is selected by default. Untick any you don't want to drive.

  2. Pick or draw a route

    Load a .gpx from your library, import one, or draw a new route on the map — click to add points, type an address to jump there, drag points to adjust. Set speed per leg and a pause at any stop, then save it with a name.

  3. Press Play

    SimSpoof walks the route point by point, telling each phone "you are here" many times a second and moving smoothly between points at your chosen speed. Pause points make it stop and wait, like a real stop. Adjust the speed factor or loop it live.

  4. Watch it react

    The dot moves on the map with a live compass and lat/lon/speed readout. Open the log & network panel to see your app receiving the new positions in real time.

Right-click the map anywhere to copy that spot's latitude/longitude — handy for building routes or checking a geofence.

Logs

  • Separate tabs for iOS system logs, Android logcat, Metro (your app's console), and SimSpoof's own status messages.
  • Auto-detected log levels with color coding and level chips to filter.
  • Search with live highlight. Type -term to exclude a term.
  • Each line carries a local timestamp. Drag to resize the area.
  • Clear button, or auto-clear on each new capture.

Network

  • API requests and responses (and websockets) — the noise is filtered out.
  • Chrome-style inspector: Headers, Payload, Preview, Response, Timing tabs.
  • Collapsible JSON, copy per tab, and text selection.
  • Pulled from Metro's HTTP logs — no system proxy, so your Mac's network stays untouched.

Performance tab

A lightweight glance at what the running app costs — without attaching a debugger.

What it shows

  • Live CPU % and memory charts for the app.
  • Time ranges: Live, 5 min, 30 min.
  • Per-process breakdown, sorted by CPU (biggest power draw first).
  • Click a process to drill into PID, threads, CPU time, uptime, memory, state — down to a per-thread table.

What it isn't

  • It's a quick glance, not a profiler. For call stacks, named threads, leaks, and real energy impact, use Xcode Instruments or the Android Studio Profiler.
  • CPU % is the power proxy — macOS doesn't expose per-process wattage without elevated tools.
  • Simulator/host processes only (the app runs natively on the Mac).

The real-iPhone catch

Worth knowing before you trust a simulator result.

Simulator — convenient, quirky

  • Some location libraries cache an old position and can look "stuck" while standing still.
  • Stop-detection (geofences) can be flaky — the simulator doesn't behave exactly like real GPS.

Real iPhone — behaves correctly

  • Real GPS hardware, real behavior.
  • SimSpoof drives it over USB or WiFi — it's just another phone you select.
  • When something has to work (a driver checking into a stop), test it here.
